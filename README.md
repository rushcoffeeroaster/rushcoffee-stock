## 📌 ภาพรวม (Overview)

Rush Coffee เป็นเว็บแอปแบบ *mobile-first* สำหรับบริหารร้านกาแฟ 2 สาขา ใช้งานได้ทันทีบนมือถือผ่านเบราว์เซอร์ ไม่ต้องติดตั้งแอป ข้อมูลทั้งหมดเก็บบน Google Sheets ผ่าน Google Apps Script และซิงค์อัตโนมัติทุกหน้า

ระบบประกอบด้วย **3 โมดูลหลัก** ที่เชื่อมถึงกันด้วยแถบเมนูนำทางร่วมด้านล่างจอ:

| โมดูล | ไฟล์ | หน้าที่ |
|---|---|---|
| 🧾 **สต๊อก** | `index.html` | จัดการสต๊อกวัตถุดิบ รับเข้า–ใช้ออก แจ้งเตือนของใกล้หมด และดูความเคลื่อนไหวล่าสุด |
| 📊 **บัญชี** | `dashboard.html` | บันทึกรายรับ–รายจ่าย ดูสรุปกำไร อัตรากำไร แนวโน้มรายวัน และเปรียบเทียบรายสาขา |
| 🧮 **ต้นทุนเมนู** | `menu-cost.html` | คำนวณต้นทุนและ GP% ของแต่ละเมนูจากสูตรวัตถุดิบ |
| ⏰ **ตอกบัตร** | `attendance.html` | ตอกบัตรเข้า-ออกงานด้วย selfie + เวลา + GPS แบบ geofence ตรวจว่าอยู่ที่ร้านจริง |
| 📈 **รายงาน** | `report.html` | สรุปชั่วโมงทำงาน · มาสาย · ลา/ไม่มา รายคน สำหรับผู้จัดการ (ล็อกด้วยรหัส) |

## 🎨 ดีไซน์ & ธีมสี (Design System)

ปรับใหม่ทั้งหมดเป็นสไตล์ **Modern Light Luxe** — พื้นหลังสว่างสะอาดตา ใช้งานง่ายกลางวัน ตัดกับงานโลหะเงิน–ทองที่ดูพรีเมียม

- 🥈 **สาขา Rush → โทนเงิน (Silver)** — เทาเมทัลลิก/แพลทินัม
- 🥇 **สาขา Rush Hour → โทนทอง (Gold)** — ทองแชมเปญ
- ⚫ **มุมมองรวม → โทนกราไฟต์ (Graphite)** — สำหรับดูภาพรวมทั้งสองสาขา

แต่ละหน้าจะเปลี่ยนสีธีม (ปุ่ม แท็บ กราฟ ไฮไลต์) ตามสาขาที่เลือกโดยอัตโนมัติ ทำให้รู้ได้ทันทีว่ากำลังดูสาขาไหน โลโก้และตัวเลขเด่นใช้เอฟเฟกต์ไล่เฉดโลหะ (metallic gradient)

**องค์ประกอบที่ปรับปรุง**

- แถบแบรนด์ด้านบน (sticky) + แถบเมนูนำทางด้านล่าง (สต๊อก / บัญชี / ต้นทุนเมนู) สลับหน้าได้ในแตะเดียว
- การ์ดมุมโค้ง เงานุ่ม เส้นบางพรีเมียม และฟอนต์ `Inter` + `Noto Sans Thai`
- การ์ดสรุปสถิติ (stat strip), แถบ gauge, ป้ายหมวดหมู่, แถบสัดส่วน (split bars) และกราฟที่ปรับสีตามธีม
- รองรับ `safe-area` ของ iPhone (notch) และทัชเป้าหมายขนาดใหญ่กดง่าย

## 🧾 โมดูลสต๊อก — `index.html`

- **แยกสาขา** Rush / Rush Hour / รวม (เพิ่มใหม่ในเวอร์ชันนี้)
- การ์ดสรุปด้านบน: จำนวนรายการทั้งหมด · ใกล้หมด · หมดสต๊อก พร้อมแบนเนอร์เตือนเมื่อมีของต้องสั่งซื้อ
- ค้นหา + กรองตามหมวด (กาแฟ / เครื่องดื่ม–ไซรัป / นม–ครีม / เบเกอรี่ / อื่นๆ)
- ปุ่ม **รับเข้า / ใช้ออก** บนการ์ด พร้อมแถบ gauge เทียบกับจุดสั่งซื้อขั้นต่ำ — รายการใกล้หมดถูกดันขึ้นบนสุดอัตโนมัติ
- แท็บ **ล่าสุด** แสดงความเคลื่อนไหวรับเข้า–ใช้ออก จัดกลุ่มตามวัน

> 💡 **หมายเหตุเรื่องการแยกสาขาในสต๊อก:** ระบบส่งค่า `branch` ไปกับทุกการบันทึกแล้ว ของเดิมที่ยังไม่มีค่าสาขาจะแสดงในทุกสาขาเพื่อไม่ให้ข้อมูลหาย หากต้องการแยกสต๊อกตามสาขาแบบสมบูรณ์ ให้เพิ่มคอลัมน์ `branch` ในชีตสต๊อก แล้วให้ Apps Script บันทึก/อ่านคอลัมน์นี้

## 📊 โมดูลบัญชี — `dashboard.html`

สรุปแบบละเอียดขึ้นกว่าเดิมมาก:

- **การ์ด Hero กำไรสุทธิ** + อัตรากำไร (margin %) เด่นชัด
- **KPI**: รายรับรวม · รายจ่ายรวม · เฉลี่ยรายรับต่อวัน · อัตรากำไร · จำนวนธุรกรรม
- **สรุปเชิงลึก (Insights)** อัตโนมัติ: วันขายดีที่สุด, หมวดรายจ่ายสูงสุด, สัดส่วนยอด Delivery และค่าคอมมิชชั่น
- **เปรียบเทียบรายสาขา** (เมื่อเลือก "รวม"): การ์ดเงิน Rush vs การ์ดทอง Rush Hour แสดงรายรับ/รายจ่าย/กำไร
- **รายรับแยกตามช่องทาง** หน้าร้าน vs Delivery แบบแถบสัดส่วน
- **กราฟแนวโน้มรายวัน** (แท่งซ้อน รายรับ + เส้นรายจ่าย) และ **โดนัทรายจ่ายแยกหมวด** พร้อมรายการเปอร์เซ็นต์
- **สรุปวัตถุดิบที่ซื้อ** รวมยอดต่อชนิด
- เลือกช่วงเวลา: รายวัน / 7 วัน / 30 วัน / ทั้งหมด และบันทึกรายรับ–รายจ่ายได้ในแท็บเดียว

## 🧮 โมดูลต้นทุนเมนู — `menu-cost.html`

- **แยกสาขา** Rush (เงิน) / Rush Hour (ทอง)
- การ์ดสรุป: จำนวนเมนู · **GP เฉลี่ย** · เมนู GP สูงสุด
- **เรียงลำดับ** ได้ตาม GP / กำไร (บาท) / ราคาขาย / ชื่อ
- กรอกสูตรวัตถุดิบหลายรายการ ระบุ *ราคาซื้อ* และ *ขนาดแพ็ก* ระบบหารต้นทุนต่อหน่วยให้อัตโนมัติ
- **ดูตัวอย่างผลคำนวณสด (Live Preview)** ขณะกรอก — เห็นต้นทุน กำไร และ GP% ทันที
- การ์ดเมนูแสดงแถบ GP% สี (แดง <30% / เหลือง 30–60% / เขียว ≥60%)

## ⏰ โมดูลตอกบัตร — `attendance.html`

ระบบลงเวลาเข้า-ออกงานสำหรับพนักงานหน้าร้าน ออกแบบมาเพื่อกันการตอกบัตรแทนกัน:

- **แยกสาขา** Rush (เงิน) / Rush Hour (ทอง)
- **3 ขั้นตอนง่ายๆ**: เลือกชื่อพนักงาน → ใส่ **PIN 4 หลัก** → **ถ่ายรูป selfie** → กด *เข้างาน* / *เลิกงาน*
- **ถ่ายรูปสด** ผ่านกล้องหน้าในเบราว์เซอร์ (ไม่ให้เลือกรูปจากแกลเลอรี) พร้อมปั๊มวันที่–เวลา–สาขา–พิกัดลงบนรูปอัตโนมัติ แล้วเก็บลง Google Drive
- **Geofence**: ต้องอยู่ในรัศมีร้าน (ค่าเริ่มต้น 150 ม.) ถึงจะตอกบัตรได้ ระบบเทียบพิกัด GPS กับตำแหน่งสาขา — ตรวจทั้งฝั่งเครื่องและฝั่งเซิร์ฟเวอร์
- **นาฬิกาสด** บนหัวจอ + การ์ดสถานะ "วันนี้ตอกเข้า/ออกล่าสุดกี่โมง"
- แท็บ **ประวัติล่าสุด** แสดงรายการพร้อมรูป เวลา ระยะห่าง และที่อยู่ (reverse geocode)

> โมดูลนี้ต้องตั้งค่า backend แยกต่างหาก (ดูหัวข้อ "ตั้งค่าระบบตอกบัตร" ด้านล่าง) เพราะต้องเก็บรูปลง Drive และมีชีตพนักงาน/PIN

## 📈 โมดูลรายงาน (ผู้จัดการ) — `report.html`

หน้าสรุปข้อมูลการทำงานจากชีต `Attendance` สำหรับผู้จัดการ เข้าได้ด้วย **รหัสผู้จัดการ 4 หลัก**:

- เลือก **สาขา** (Rush / Rush Hour / รวม) และ **ช่วงเวลา** (สัปดาห์นี้ / เดือนนี้ / กำหนดเอง)
- **KPI**: ชั่วโมงทำงานรวม · จำนวนพนักงานที่มาทำงาน · มาสาย (ครั้ง) · ลา · ไม่มาตอก (โดยประมาณ)
- **การ์ดรายคน**: ชั่วโมงรวม จำนวนวันที่มา ป้ายสาย/ไม่ตอกออก/ลา — แตะเพื่อกางดูรายวัน (เวลาเข้า-ออก + ชั่วโมง + ธงมาสาย)
- คำนวณชั่วโมงจากการจับคู่ "เข้า→ออก" ของแต่ละวัน, ตัดสิน "มาสาย" จากเวลาเข้างานที่ตั้งไว้
- ใช้ Apps Script + ชีตตัวเดียวกับระบบตอกบัตร (ไม่ต้อง deploy เพิ่ม)

> ใช้งานได้จริงต้องตั้ง `MANAGER_PIN` และเวลาเข้างาน (`SHIFT`) ในไฟล์ `attendance.gs` — ดูด้านล่าง

## 🚀 การติดตั้ง & Deploy

1. **อัปโหลดไฟล์** ทั้งหมด (`index.html`, `dashboard.html`, `menu-cost.html`, `README.md`, `.nojekyll`) ขึ้น branch `main`
2. เปิด **Settings → Pages** เลือก Source = `main` / root แล้วบันทึก
3. เปิดเว็บที่ `https://<username>.github.io/rushcoffee-stock/`
4. **เชื่อม Backend:** แก้ค่า `API_URL` ในแต่ละไฟล์ให้เป็น URL `/exec` ของ Google Apps Script ของคุณ (ค่าปัจจุบันชี้ไปยัง backend เดิมอยู่แล้ว)

> หน้าสต๊อกใช้คนละ Apps Script กับหน้าบัญชี/ต้นทุนเมนู — ตรวจให้ `API_URL` ในแต่ละไฟล์ตรงกับชีตที่ต้องการ

## ⏰ ตั้งค่าระบบตอกบัตร (Attendance Setup)

โค้ดหลังบ้านอยู่ในไฟล์ **`attendance.gs`** — ทำตามนี้ครั้งเดียว:

1. **สร้าง Google Sheet** ใหม่ 1 ไฟล์ ตั้งชื่อเช่น `Rush Attendance`
2. ในชีตนั้นไปที่ **Extensions → Apps Script** ลบโค้ดเดิม แล้ววางเนื้อหาจาก `attendance.gs` ทั้งหมด
3. **แก้ค่า CONFIG** ด้านบนของไฟล์:
   - `BRANCHES` → ใส่พิกัด GPS จริงของแต่ละสาขา (ใน Google Maps คลิกขวาที่จุดร้าน → คัดลอกเลข lat, lng)
   - `RADIUS_M` → รัศมีที่ยอมให้ตอกบัตร (แนะนำ 120–200 ม.)
4. รันฟังก์ชัน **`setup`** หนึ่งครั้ง (กดอนุญาตสิทธิ์ตอนถาม) — ระบบจะสร้างชีต `Employees`, `Attendance` และโฟลเดอร์เก็บรูปให้อัตโนมัติ จากนั้นเปิด **Executions/Log** คัดลอก ID โฟลเดอร์ที่ขึ้นมาไปวางในตัวแปร `PHOTO_FOLDER_ID` (ทำให้รูปไปอยู่โฟลเดอร์เดียวเสมอ)
5. **ใส่รายชื่อพนักงาน** ในชีต `Employees`: คอลัมน์ `id`, `name`, `branch` (`rush`/`rushhour`), `pin` (4 หลัก), `active` (`yes`/`no`)
   - ⚠️ ตั้งคอลัมน์ `pin` เป็นรูปแบบ **ข้อความ (Plain text)** เพื่อไม่ให้เลข 0 นำหน้าหาย
6. **Deploy → New deployment → ⚙️ → Web app**
   - Execute as: **Me**
   - Who has access: **Anyone**
   - กด Deploy แล้วคัดลอกลิงก์ที่ลงท้าย `/exec`
7. เปิดไฟล์ **`attendance.html`** วางลิงก์นั้นในตัวแปร `API_URL` แล้วอัปโหลดขึ้น GitHub

**หมายเหตุ**

- PIN ตรวจสอบฝั่งเซิร์ฟเวอร์ ไม่ถูกส่งมาเก็บในเว็บ (หน้าเว็บดึงมาแค่รายชื่อ)
- รูป selfie ถูกตั้งสิทธิ์เป็น *ใครมีลิงก์ก็ดูได้* เพื่อให้แสดงในหน้าประวัติได้ — เป็นข้อมูลภายใน เก็บในโฟลเดอร์ Drive ของคุณเอง
- ต้องเปิดผ่าน **HTTPS** (GitHub Pages เป็น https อยู่แล้ว) กล้องและ GPS ถึงทำงาน และผู้ใช้ต้องกด *อนุญาต* กล้อง/ตำแหน่ง

**เปิดใช้หน้ารายงานผู้จัดการ (`report.html`)** — ใช้ Apps Script ตัวเดียวกับตอกบัตร

1. เปิดไฟล์ `report.html` วางลิงก์ `/exec` เดียวกันในตัวแปร `API_URL`
2. ในไฟล์ `attendance.gs` ตั้ง **`MANAGER_PIN`** เป็นรหัสผู้จัดการ (ค่าเริ่มต้น `1357` — ควรเปลี่ยน) แล้ว Deploy ทับเวอร์ชันเดิม
3. ตั้งเวลาเข้างาน + ผ่อนผัน (ใช้ตัดสิน "มาสาย") ที่ `SHIFT` — ถ้าสองสาขาเข้างานคนละเวลา แก้ที่ `SHIFT_BY_BRANCH`
4. (ออปชั่น) บันทึกวันลา: สร้างชีตชื่อ **`Leave`** คอลัมน์ `employeeId`, `date` (YYYY-MM-DD), `type`, `note` — ระบบจะแสดงเป็น "ลา" และไม่นับเป็นไม่มา
5. การคำนวณ "ไม่มาตอก" เป็นค่าโดยประมาณ (วันในช่วงถึงวันนี้ − วันที่มา − วันลา) เหมาะกับร้านที่เปิดทุกวัน หากมีวันหยุดประจำให้ลงในชีต Leave

## 🗂️ โครงสร้างข้อมูล (Data Model)

**สต๊อก** — `getAll → { items, logs }`

- `items`: `id`, `name`, `unit`, `category`, `qty`, `min`, `branch`
- `logs`: `itemName`, `type` (`in`/`out`/`add`), `delta`, `note`, `time`, `branch`
- actions: `getAll`, `saveItem`, `moveStock`, `logOnly`, `deleteItem`

**บัญชี** — `getAll → { income, expense }`

- `income`: `id`, `date`, `channel` (`store`/`delivery`), `amount`, `note`, `branch`
- `expense`: `id`, `date`, `category` (`stock`/`rent`/`commission`/`other`), `amount`, `note`, `branch`, `itemName`, `itemQty`, `itemUnit`
- actions: `getAll`, `addIncome`, `addExpense`, `deleteIncome`, `deleteExpense`

**ต้นทุนเมนู** — `getAll → { recipes }`

- `recipes`: `id`, `menuName`, `sellingPrice`, `ingredientName`, `ingredientQty`, `ingredientUnit`, `buyPrice`, `packSize`, `branch`
- actions: `getAll`, `addRecipeRow`, `deleteRecipeMenu`

**ตอกบัตร** (Apps Script แยก + Sheet แยก)

- `Employees` (ตั้งค่าเอง): `id`, `name`, `branch`, `pin`, `active`, `shiftStart` (กะ เช่น `10:00`/`12:00`; เว้นว่าง = ใช้เวลาเข้างานตามวันของสาขา)
  - เวลาเข้างานรายสาขา/รายวันตั้งใน `attendance.gs` (`SHIFT_BY_BRANCH`) — เช่น Rush จ–พฤ 07:00 / ศ 07:00 / ส–อา 08:00; Rush Hour มี 2 กะ (10:00, 12:00) แยกรายคนที่ `shiftStart`
- `Attendance` (ระบบเขียน): `id`, `timestamp`, `dateLocal`, `employeeId`, `employeeName`, `branch`, `type` (`in`/`out`), `lat`, `lng`, `distanceM`, `accuracyM`, `address`, `photoUrl`, `clientTime`
- `Leave` (ออปชั่น สำหรับรายงาน): `employeeId`, `date`, `type`, `note`
- actions: `getConfig`, `punch`, `getRecords`, `getReport` (รายงานผู้จัดการ — ต้องมี `managerPin`)
